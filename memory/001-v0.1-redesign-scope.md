# v0.1 Whole-Project Redesign Scope

## Decision

`v0.1` is the first whole-project redesign milestone for this repo.

The scope is not limited to the landing or auth experience. It covers the shared shell and every major product area that a user can see after login.

## Design stance

- follow `DESIGN.md` strictly
- borrow visual character from `../landing-page`
- do not copy raw implementation details from `../landing-page` when they violate `DESIGN.md`
- prefer shared tokens and reusable primitives over page-by-page custom styling

## Product areas included

- auth and account entry flows
- shared navigation and layout shell
- dashboards and test-library browsing
- review and score analysis
- test-taking
- vocab board and flashcards
- admin fix board and admin forms
- parent dashboard
- settings and hall-of-fame

## Implementation stance

- keep business logic and data flows intact wherever possible
- redesign presentational layers first
- extract shared UI primitives when repeated patterns emerge during the redesign
- preserve responsive behavior and explicitly design for dense content and long text

## Important constraint

The sibling `../landing-page` currently includes blur and raw hex usage in places.

For this repo, those are reference artifacts, not rules. When the sibling project conflicts with `DESIGN.md`, follow `DESIGN.md`.

## Recommended execution order

1. global tokens and utility classes
2. shell and navigation
3. auth surfaces
4. student dashboard and library pages
5. review, vocab, and fix
6. live test-taking
7. parent, settings, hall-of-fame, and admin
8. final consistency sweep
